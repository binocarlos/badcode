# From-scratch rework of Camping & Karen — design

**Date:** 2026-06-27
**Status:** approved (design); pending implementation plan
**Owner:** Kai

## Problem

Camping and Karen were imported from the old Storyteller setup. Each has solid
story canon in `docs/<story>/`, a fully-rendered comic in
`apps/web/src/comics/<story>/` (backed by GCS bucket assets), and a song stub.
But the **art was imported, not generated by our pipeline** — so there are no
Flow character sheets and no per-panel prompt-record provenance (the only
exceptions are Karen's two recently-animated slides, `anim/a12` and `anim/a13`).

Without provenance and pipeline-owned characters, "just like that, but change X"
is not a cheap step — every revision means re-sourcing art by hand. We want both
stories to live on the official `make-comic` pipeline, faithful to the existing
narrative and characters, with brand-new pipeline-owned art and full provenance.

## Goal

Produce **parallel** `-v2` versions of each story, rebuilt from scratch through
the official pipeline, while the original imported versions stay frozen as
reference.

- **End state:** full from-scratch art regeneration — new Flow character sheets
  **and** every panel image (and, in a deferred phase, animations), each with a
  prompt-record so revisions are one cheap step.
- **Faithful, not reinvented:** same premise, political point, characters, and
  beat sequence as the imported originals. New art, same story.

## Decisions (locked)

1. **End state = full from-scratch art regen.** Not "reuse art + add
   provenance"; we regenerate character sheets and panel images via the Flow loop.
2. **Sequence = Camping first, then Karen.** Camping has only two leads and a
   tighter arc, so it de-risks and tunes the from-scratch loop before we take on
   Karen's larger cast and 53 panels.
3. **Parallel structure = new `-v2` slug.** Originals are never edited.
   - `docs/camping-v2/`, `apps/web/src/comics/camping-v2/`, bucket
     `comics-v2/camping-v2` (and the same for `karen-v2`).
4. **Isolation = git worktree/branch** (e.g. `feat/camping-v2-rework`), because
   commits in this repo auto-push (publishing). The rework does not happen
   directly on `main`.
5. **Style choice is deferred** to the character-image stage (match the
   original's look vs. apply the "clean-cinematic" creative direction) — decided
   when we see the first Flow portrait, not up front.

## Approach

The imported comics are a head start for the pipeline: instead of *ideating* a
story (make-comic Stage 1), we **reverse-engineer canon** from the existing
`docs/<story>/` + `comic.json`. The user's two-stage framing maps onto the
official six-stage `make-comic` workflow:

| User framing | make-comic stages | Produces |
|---|---|---|
| **Stage 1 — capture essence** | 1 (story), 2 (characters), 4 (storyboard) | `docs/camping-v2/` canon: `story.md`, `characters/*.md`, `storyboard/index.md` + `pNN.md` |
| **Stage 2 — rework via Flow** | 3 (character images), 5 (panel images), 6 (assemble) | Flow character sheets + every panel image + the rendered `camping-v2` comic |

This is a composition of existing skills (`make-comic`, which itself composes
`new-story` + Flow + `@badcode/comic`; then `animate-slide`). It introduces no
new tooling — only a reverse-engineering input step and a deferred animation
phase.

## Phase structure (Camping — the de-risking run)

### Phase 1 — Essence capture (canon, no art)
Read `docs/camping/` + the imported `comic.json` and distil into `docs/camping-v2/`:

- `story.md` — premise, political point, the ~55-beat spine, faithful to the
  original. **Open decision to surface:** Camping's ending is flagged "open" in
  the original README; resolve it explicitly with the user here.
- `characters/{tarquin,bob}.md` — official character canon **with visual `sheet:`
  descriptions** (originals are bare). Include the two object-refs (tent, car).
- `storyboard/index.md` + one `pNN.md` per panel — each carrying the existing
  caption/bubble text plus a planned image prompt.

Pure read + write-docs. Fully reviewable before any image is generated.

### Phase 2 — Character images via Flow
Generate a Flow Character + portrait + prompt-record for Tarquin and Bob (plus
object-refs for the tent and car). This is where the **official characters** come
into being and where the **look is locked**. Style decision happens here.

### Phase 3 — Panel images via Flow
Regenerate each static panel against the locked characters: one image + record
per `pNN.md`. Provenance complete.

### Phase 4 — Assemble
Build `apps/web/src/comics/camping-v2/` against `@badcode/comic`
(`AUTHORING.md` is mandatory reading), wire the manifest + route, verify it
renders in the browser.

### Phase 5 — Animations (deferred)
The 13 Camping animations are **not** regenerated inline. Once camping-v2 renders
as a static comic on the bucket pipeline, apply the existing `animate-slide`
skill panel-by-panel. Keeps the loop simple and reuses proven tooling.

## Karen (after Camping's loop is proven)

Identical phase structure. Deltas only:

- **Larger cast:** Karen, Ari, the egging-on colleague, the President, and the AI
  (working name "Sean/Shaun AI" — name to confirm; it is an open thread in the
  original canon). Cross-panel character consistency is the main new challenge.
- **53 panels** across 6 acts (incl. the 100-years-later lecture-hall frame).
- Karen already has two Flow-provenanced animations (a12, a13) we can lean on as
  reference for the animate-slide phase.

## Guardrails

- **Gated, agent-driven.** `make-comic` and `animate-slide` are gated skills:
  every *produce* step waits for user approval. No silent batch generation of
  ~33 images.
- **Flow needs a logged-in browser.** Runs over the user's session; the user may
  need to be present to drive/auth it.
- **Originals frozen.** Nothing under `docs/camping/`, `docs/karen/`,
  `apps/web/src/comics/{camping,karen}/`, or `comics-v2/{camping,karen}` is
  edited.

## Out of scope

- Songs (the `suno-prompt` track work is tracked separately per story).
- Any change to the imported originals.
- New tooling — this is pure composition of existing skills.

## Success criteria

- `docs/camping-v2/` is a complete, faithful canon (story + characters with
  `sheet` descriptions + per-panel storyboard) reviewable independently of art.
- Every Camping character and panel has a Flow-generated asset **and** a
  prompt-record; "change X" is one cheap regen step.
- `camping-v2` renders in the browser via `@badcode/comic` on the bucket
  pipeline.
- The same is achieved for `karen-v2` using the proven loop.
